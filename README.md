# Leetcode

This repository is my personal practice space for solving LeetCode and DSA problems in Java. It describes what I am doing: building logic, learning approaches, and training myself to solve algorithmic challenges clearly and efficiently.

## What I am doing

- Solving problems step by step, starting from a simple approach and improving it with better logic.
- Writing code to handle edge cases, input variations, and real algorithm constraints.
- Practicing common patterns like two pointers, binary search, prefix sums, sliding windows, and sorting-based solutions.
- Using each problem to strengthen my understanding of data structures and how they affect performance.

## Why I am doing this

- To build strong reasoning for problem solving, not just memorizing answers.
- To learn how to convert a problem statement into a working algorithm.
- To practice writing clean Java code with clear variable names and straightforward logic.
- To prepare for coding interviews and DSA assessments by solving many different question types.

## How I build logic

- Read the problem carefully and identify the goal.
- Work through examples manually to understand the input/output relationship.
- Start with a basic solution first, then find ways to optimize it.
- Pay attention to time complexity and space complexity as I improve the approach.
- Test the solution with edge cases before finalizing it.

This repo is a record of my learning process: the problem-solving approach, the logic I build, and the Java solutions I create while practicing DSA and LeetCode-style questions.
